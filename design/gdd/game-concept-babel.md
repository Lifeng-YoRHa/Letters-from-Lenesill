## 概述

在巴别塔诞生之后的虚拟世界中，语言已经破碎。玩家扮演一名信使，穿越被神使封锁的路径，将承载着语义信息的卡牌递送给收件人。每张卡牌同时拥有战斗数值和语义含义——当你打出一张卡对抗神使时，它就永远失去了作为信息载体的价值。

## 玩家幻想

玩家幻想自己是破碎世界中的桥梁建造者。每一次递送都是在混乱中建立连接，每一次牺牲都是在为更大的理解付出代价。最终目标不是征服，而是理解——重构巴别塔崩塌前的原初语言。

## 核心循环

### 30秒循环（Moment-to-Moment）

神使设置牌型门槛 → 玩家从包裹中打出卡牌试图压过它 → 每张打出的卡永久离开包裹 → 核心张力：过关 vs. 保留语义素材

**是否内在有趣？** 是的，前提是代价被真切感受到。玩家烧掉一张卡时，需要看到那条未送达的信息在收件人那边产生了什么空洞。视觉/音频反馈（未读信件溶解、收信人等待动画中断）比单纯的数值损失更有情感重量。

### 5分钟循环（一次完整递送旅程）

```
遭遇神使（打牌闯关，卡牌离开包裹）→ [随机2-3个站点] → 抵达收件人（用剩余卡牌语义拼凑指定信息）
```

站点池：回想（回收部分战斗卡牌）/ 拾遗（沿途事件改变包裹内容）/ 工坊（获取辅助道具）/ 遗忘（销毁无用卡牌）。每次递送随机抽取2-3个，顺序和组合不可预测。

**再来一局钩子**：这次为了打败神使的对子要求烧掉了「红桃A」，抵达时发现它的语义是「爱」——而指定信息恰好需要「爱」。下次能不能用更小的牌过关？

**本层选择**：每张卡打或不打、回想时回收哪些、拾遗事件中冒不冒险、工坊买不买道具、遗忘哪张。

### 单次游玩循环（30–120分钟）

一次运行 = 从枢纽到终点的长程递送，包含多个神使关卡和中间阶段。

- **完整流程**：从枢纽出发 → 携带初始信件集 → 沿途收集新信件/失去信件 → 抵达区域终点交付 → 决定哪些「卡牌—语义」对应关系带入下一次运行
- **自然停止点**：完成一条递送路线后、进入新区域前
- **离线钩子**：那些没送出去的信、那些你选择了战斗而牺牲的信息——玩家会在关闭游戏后回想「如果我当时……」

### 长期进度循环（天/周）

- **成长方式**：Knowledge + Options
  - 知识：理解不同收信人的语言碎片和反应模式
  - 选项：解锁新语义类别、信使类型、递送路线
  - 能力：有限的包裹容量增长、更好的信息保存手段
- **终极目标**：重构巴别塔崩塌前的原初语言，或选择让碎片世界保持分离
- **游戏何时算「通关」**：完成若干章节后抵达语言统一/分离的终局

## MDA分析

| 维度 | 设计目标 |
|------|----------|
| Mechanics（机制） | 双用卡牌牺牲系统、扑克牌型对抗、随机站点序列、语义拼凑 |
| Dynamics（动态） | 资源稀缺性迫使玩家在战斗与通信之间做零和选择；语义流变要求玩家每局重新学习语言规则 |
| Aesthetics（美学） | Discovery（发现新语义映射）、Narrative（涌现式角色互动）、Challenge（最小代价通关的优化空间）|

## 玩家动机档案（SDT）

- **自主性 (Autonomy)**：高——玩家完全决定送什么、给谁、按什么顺序、牺牲哪张
- **胜任感 (Competence)**：通过模式识别成长（学习收信人反应规律、优化递送路线、记住有效语义映射）
- **关联性 (Relatedness)**：强——通过发送/接收消息与角色建立连接，通过揭开世界碎片历史与虚拟世界建立情感纽带

## 心流状态设计

- **挑战与技能平衡**：早期运行高模糊度（高挑战），随着玩家锁定更多语义映射，挑战曲线通过引入新区域和更复杂的神使要求来维持
- **清晰目标**：每次运行开始时明确指定需要递送的核心信息，玩家始终知道最终目标
- **即时反馈**：打出卡牌时立即看到战斗结果和语义损失；抵达收件人时看到拼凑出的信息效果

## 游戏支柱

### 支柱一 — Every Card is a Choice
> 包裹中的每张卡必须服务于战斗或通信之一；用于一方就永久失去另一方的价值。
> **设计测试**：如果我们在争论是否让玩家免费回收打出的卡牌，这根支柱说不行——回收必须通过特定站点获取，且带有机会成本。

### 支柱二 — Meaning is Discovered, Not Given
> 叙事意义从玩家选择、卡牌语义与AI生成的世界反应的交汇处涌现，而非来自人工编写的分支路径。
> **设计测试**：如果我们在争论写一段脚本过场还是让角色反应从投递的卡牌语义中涌现，这根支柱支持涌现。

### 支柱三 — Meaning is Fluid, Memory is Fragile
> 每张卡的语义含义在每次运行间流动变化，迫使玩家重新学习这门语言；但部分对应关系可以被锁定，成为跨越周目的脆弱记忆锚点。
> **设计测试**：如果我们在争论是否从开局就固定所有卡牌语义，这根支柱说不——初始模糊是必需的，选择性保留是 earned reward。

### 支柱四 — Language is Power
> 理解卡牌语义如何与世界的隐藏语言规则交互，是玩家成长的主要形式。
> **设计测试**：如果我们在争论是否给出明确的语义提示还是让玩家通过实验发现交互，这根支柱支持保留信息差。

## Anti-Pillars

1. **我们不会做传统数值化角色成长**（力量应来自知识和选项，而非数字）
2. **我们不会做人工编写的分支叙事树**（我们要的是涌现，不是Choose-Your-Own-Adventure）
3. **我们不会允许玩家在运行前预览所有站点类型**（不可预测性必须被保护）
4. **我们不会做无后果的「安全模式」测试**（牺牲必须始终真实）

## 玩家类型验证

- **核心受众**：Explorers（探索语义映射与世界规则）+ Storytellers（从消息投递中编织叙事）
- **次要受众**：Achievers（优化递送路线、最小化损耗）
- **不适合**：追求快节奏操作的Competitors、期待保证情感高潮的电影化叙事玩家、寻求多人体验的Socializers
- **市场参照**：Inscryption、Balatro、Loop Hero、Cultist Simulator

### MVP（6–8周）

- 1个区域
- 1种神使门槛（如对子）
- 2种站点（工坊 + 回想）
- 3–5组手编语义映射（**无AI**，先验证核心机制）
- 1位收件人，基础反应
- 核心牺牲机制可运行

### 现实交付版（4–6月）

- 手编模板 + 程序变体
- 3区域、5角色
- 全4种站点
- 1–2结局
- Steam发布

### 完整愿景（9–12月）

- AI生成语义
- 5区域、10+角色
- 多结局
- 完整Steam发布

### 最大风险

1. **技术风险**：首款游戏就引入AI生成语义，输出品控和一致性是未知数
2. **设计风险**：双用牺牲机制可能从「有意义的痛苦」滑向「单纯惩罚玩家」
3. **市场风险**：涌现叙事成败高度依赖执行质量
4. **范围风险**：首款游戏 × AI集成 × 叙事野心 = 经典过度扩张三角

## 视觉身份锚点（Visual Identity Anchor）

**选定方向**：Digital Relic / 数字遗迹

**核心视觉规则**：所有视觉元素都应同时传达「古老语言的神秘感」和「数字世界的破碎感」——像是出土的数据碎片。

**支撑原则**：

1. **碎片即完整** — 卡面设计有意保留缺失、故障、不完整的视觉元素
   - **设计测试**：如果一个UI元素看起来「太干净」，它就不符合这个方向
2. **语义可视化** — 卡牌语义通过视觉符号（而非纯文字）表达
   - **设计测试**：玩家应能在不读文字的情况下大致猜出一张卡的语义类别
3. **动态腐朽** — 被消耗/遗忘的卡牌应有视觉老化/数据损坏的过程
   - **设计测试**：打出一张卡时，玩家应看到它从「完整信息」变为「碎片」

**色彩哲学**：高对比度的暗底（深紫/墨黑）+ 荧光文字色（青绿/琥珀），强调「在黑暗中寻找光亮」的阅读体验
