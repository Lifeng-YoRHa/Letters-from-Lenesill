# 第一章地图生成规则

## 核心原则

- **特殊节点固定**：委托×1、安全屋×2、黑市×2，在每版地图中的**层位置、索引位置、相互连接**固定不变。
- **其他节点随机**：普通战斗、艰难战斗、突发事件、废墟、道路，填充剩余槽位，类型每次冒险从随机池中抽取。
- **结构约束固定**：每版地图的核心拓扑特征（死路比例、陷阱机制、回环规则等）由生成算法保证，不依赖特定节点的类型。

## 通用随机池（34个中间节点）

| 类型 | 数量 | 说明 |
| :--- | :-: | :--- |
| 委托 | 1 | **固定节点**，不参与随机 |
| 安全屋 | 2 | **固定节点**，不参与随机 |
| 黑市 | 2 | **固定节点**，不参与随机 |
| 普通战斗 | 8 | 随机池 |
| 艰难战斗 | 4 | 随机池 |
| 突发事件 | 8 | 随机池 |
| 废墟 | 3 | 随机池 |
| 道路 | 6 | 随机池 |

**随机池总计**：8+4+8+3+6 = 29个节点，填充到每版地图的29个非特殊槽位中。

| 位置 | 连接约束 | 类型来源 |

### 生成约束

1. **保证连通**：从 START 出发，必须经过固定节点链，最终可达 BOSS。
2. **死路比例**：约 20% 节点度数为 1（走到头必须原路返回）。
3. **同层簇状**：同层连接呈 2~3 节点小簇，不贯穿全层。
4. **枢纽集中**：少数节点度数为 3~4，多数节点度数为 1~2。
5. **跨层跳跃**：允许 Layer i 直接连 Layer i+2 的捷径。

### 固定服务链

```
START -> ... -> L2_3(黑市1) -> L3_2(安全屋1) -> L3_3(委托) -> ... -> L5_2(黑市2) -> L5_4(安全屋2) -> ... -> BOSS
```

特殊节点之间形成一条"服务链"，玩家在任何一次冒险中都能依赖这条链找到补给和任务。

## 呈现规则（已确认）

### 节点可见性

- **位置也隐藏**：玩家只能看到**当前所在节点**和**与其直接相连的相邻节点**；远处节点完全不可见（不显示位置、不显示连线）。
- 踏入节点后，该节点位置永久揭示，但类型只在踏入后显示。
- "手电筒"消耗品可以揭示随机 2 个节点的**位置+类型**，无论距离远近。

### 视觉布局

- 每层节点在屏幕上**水平分布**（Layer 1 在最左，Boss 在最右）。
- 同层节点在垂直方向上**略微浮动**（每次生成时 Y 坐标有随机偏移，不排成严格的直线）。
- 层与层之间的水平间距固定，但具体节点的 X 坐标允许在小范围内随机偏移。
- 连线用直线或轻微曲线连接相邻节点。

### 区域污染源特殊处理

- 逃离 Boss 战是**纯游戏逻辑**，不在地图拓扑中添加额外连线。
- 触发逃离后，玩家节点直接跳转到**最近的安全屋**（计算拓扑距离最短的安全屋节点）。
- Boss 的已损失生命值恢复一半，由战斗系统处理，与地图数据无关。

## 运行时生成流程

1. 选择地图版本（原版/漏斗/回环）。
2. 读取该版本的层分布和固定节点布局。
3. 将 5 个特殊节点放入固定槽位。
4. 从随机池中抽取 29 个类型， shuffle 后填入剩余槽位。
5. 根据该版本的结构性约束，建立固定连接。
6. 在约束范围内，随机生成非结构性节点之间的连接。
7. 为每个节点生成带浮动的屏幕坐标（X：层基准+随机偏移；Y：层内均匀分布+随机偏移）。
8. 验证：START 到 BOSS 是否连通；死路比例是否符合；特殊节点链是否完整。
